# 2026-03-25

- `/v2/gateway/discovery` 已修复“监听 `0.0.0.0` 返回不可达 host”问题：返回优先级为 `--advertise-host/NEPTUNE_ADVERTISE_HOST` > 请求 `Host` 头 > server hostname（若仍为通配地址则回退 `127.0.0.1`）。
- `neptune-gateway serve` 新增 `--advertise-host` 参数，并透传到 `NeptuneGatewaySwift.makeApplication(...)` 与路由运行时配置。
- 新增回归测试：
  - `testDiscoveryEndpointUsesAdvertiseHostOverride`
  - `testDiscoveryEndpointUsesRequestHostHeaderWhenListeningOnAllInterfaces`
  并扩展 CLI 参数测试覆盖 `--advertise-host`。
- 本机验证通过：
  - `swift test` 全量通过（39 passed, 1 skipped）
  - Harmony 冒烟下 discovery 不再返回 `0.0.0.0`，日志显示 `source=manual-dsn host=10.0.2.2 port=18765`。
- 现状结论：模拟器环境仍优先命中 manual DSN，mDNS 在 Harmony 模拟器内尚未命中；但 discovery 返回地址已可达，WS 与 ingest 链路可用。
- `/v2` 命令下发已抽象为 `GatewayMessageBus + ClientTransportAdapter`：Gateway 内部统一使用 `BusEnvelope/BusAck`，HTTP callback / usbmuxdHTTP 走同一套分发与 ACK 归并逻辑，WebSocket adapter 先保留占位实现以便后续接入在线 SDK socket。
- `POST /v2/clients:register` 已收敛为 `callbackEndpoint(必填) + preferredTransports + usbmuxdHint`；旧 `commandUrl`、`callbackBaseUrl + callbackPath` 别名不再接受，`preferredTransports` 包含 `usbmuxdHTTP` 时强制校验 `usbmuxdHint`。
- 新增单元/路由测试覆盖：适配器选择与回退、ACK 聚合、register 字段校验，以及 usbmuxd 报文字节序、握手成功/失败路径。
