# 2026-03-24

- `GatewayStore` 已改为 SQLite 持久化主存储，默认使用独立的临时数据库文件；调用方可通过 `storageURL` 指定固定路径以跨实例保留数据。
- 保留策略已实现并可配置：默认 `maxRecordCount=200000`、`maxAge=14d`。
- `/v2/logs` 的过滤、`afterId + waitMs` 轮询，以及 ingest 的单条 JSON、数组 JSON、NDJSON 写入行为保持兼容。
- CLI 代理命令未改动，测试已覆盖相关解析与路由行为。
- `GatewayStore` 已从手写 `SQLite3` C API 迁移到 GRDB，继续复用原有 schema 与查询语义，避免回退 retention、`waitMs`、JSON/NDJSON ingest 与 CLI 代理能力。
- 新增路由回归测试，覆盖 `afterId` 直接过滤，以及 retention 后 `/v2/sources` 清理孤儿 source 的行为。
- 新增默认跳过的性能门禁测试，显式启用后会并发 ingest 100,000 条记录并校验查询可用、ID 无重复且严格递增；同时提供 `scripts/perf_gate.sh` 作为一键入口。
- `/v2/logs?format=text` 已实现为 `timestamp<TAB>level<TAB>platform<TAB>message` 的逐行文本输出，并补充对应路由测试。
- `afterId + waitMs` 长轮询已从固定 sleep 重试改为 `GatewayStore` actor 内 waiter/continuation 通知；新记录入库后按最新 ID 唤醒等待方，超时后仅重查一次。
- 新增 `scripts/build-cli-release.sh`（发布构建 + sha256 + manifest）与 `BuildCLIReleaseScriptTests`；README 补齐 CLI 分发与 iOS/Android/Harmony 代理用法说明，`dist/` 产物目录加入忽略规则。
