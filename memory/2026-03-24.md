# 2026-03-24

- `GatewayStore` 已改为 SQLite 持久化主存储，默认使用独立的临时数据库文件；调用方可通过 `storageURL` 指定固定路径以跨实例保留数据。
- 保留策略已实现并可配置：默认 `maxRecordCount=200000`、`maxAge=14d`。
- `/v2/logs` 的过滤、`afterId + waitMs` 轮询，以及 ingest 的单条 JSON、数组 JSON、NDJSON 写入行为保持兼容。
- CLI 代理命令未改动，测试已覆盖相关解析与路由行为。
- `GatewayStore` 已从手写 `SQLite3` C API 迁移到 GRDB，继续复用原有 schema 与查询语义，避免回退 retention、`waitMs`、JSON/NDJSON ingest 与 CLI 代理能力。
- 新增路由回归测试，覆盖 `afterId` 直接过滤，以及 retention 后 `/v2/sources` 清理孤儿 source 的行为。
- 新增默认跳过的性能门禁测试，显式启用后会并发 ingest 100,000 条记录并校验查询可用、ID 无重复且严格递增；同时提供 `scripts/perf_gate.sh` 作为一键入口。
- `/v2/logs?format=text` 已实现为 `timestamp<TAB>level<TAB>platform<TAB>message` 的逐行文本输出，并补充对应路由测试。
- `afterId + waitMs` 长轮询已从固定 sleep 重试改为 `GatewayStore` actor 内 waiter/continuation 通知；新记录入库后按最新 ID 唤醒等待方，超时后仅重查一次。
- 新增 `scripts/build-cli-release.sh`（发布构建 + sha256 + manifest）与 `BuildCLIReleaseScriptTests`；README 补齐 CLI 分发与 iOS/Android/Harmony 代理用法说明，`dist/` 产物目录加入忽略规则。
- CLI 发布正式入口已迁移到 GitHub Release：新增 `.github/workflows/release-cli-tag.yml`，支持 `v*` tag push 和手动 `workflow_dispatch`，复用 `scripts/build-cli-release.sh` 生成二进制/sha256/manifest，并通过 `softprops/action-gh-release` 上传到 Release；README 已改为 tag 发布说明，并补了 workflow 配置测试。
- 发布 workflow 进一步收紧：`workflow_dispatch` 的 `tag_name` 现在会按 `^v[0-9]+\.[0-9]+\.[0-9]+(-[0-9A-Za-z.-]+)?$` 先做校验，不合法直接失败；同时用 `git log` 从最近 tag 生成简要 changelog 片段并预置到 release body，原有 `dist/cli-release/*` artifacts 上传保持不变。
- `/v2/ws` 已恢复为 JSON 文本帧协议：支持 `hello`、`heartbeat` 45s 超时、`command.send`/`command.dispatch`/`command.ack`、`event.command_ack`、`event.command_summary`，并保留 `event.log_record(topicId=101)` 转发。
- WS 协议测试采用短超时注入配置以避免长等待；全量 `swift test` 已通过，当前测试总计 33 项，1 项性能门禁默认跳过。
- Gateway CLI 已接入 mDNS 广播（`_neptune._tcp.local.`），`serve` 新增 `--mdns/--no-mdns`、`--mdns-service-name`、`--mdns-service-type`、`--mdns-domain`；默认从 `NEPTUNE_MDNS_*` 读取并开启广播。
- 修复了 `serve` 参数与 Vapor 二次解析冲突：启动时使用净化后的 environment 参数，避免 `--mdns*` 被误判为未知输入。
- 本机已完成 mDNS 冒烟：`dns-sd -B _neptune._tcp local.` 可发现实例，`dns-sd -L <name> _neptune._tcp local.` 可解析到 `linhey.local:<port>` 与 `version` TXT 记录。
- 当前现状：iOS 与 Android 已有 mDNS 发现实现；Harmony 端仍是空实现（仅 manual DSN 回退），因此 Harmony 模拟器场景仍需手动 DSN/端口映射策略兜底。
